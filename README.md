# СКЛАД · забалансовый учёт ТМЦ для Битрикс24

Интерактивный **макет** приложения складского забалансового учёта на базе Битрикс24.
Решение не заменяет учёт в 1С, а ведёт **забалансовый учёт** оборудования и аксессуаров,
переданных по актам приёма-передачи (АПП): движение ТМЦ между складами, инвентаризация
по QR/штрих-коду, акты списания и аналитика по гарантийным срокам.

> Демонстрационный прототип с моковыми данными. Дизайн — «стеклянно-металлический»
> (Chromify-стиль): серебристо-серый фон, объёмные металлические элементы, мягкие тени,
> зелёный glass-акцент.

## Экраны

| Раздел | Что показывает |
|---|---|
| **Обзор** | KPI забаланса, валовые остатки по номенклатуре, журнал перемещений, быстрые действия |
| **Справочник** | Родительские позиции → отдельные единицы (инв./зав. номер, штрих-код, гарантия, производитель, стоимость, дефекты). Карточка ТМЦ |
| **Склады** | Ячейки с договором-основанием (АПП), ответственным, несгораемым остатком; клонирование, загрузка со скана |
| **Инвентаризация** | Сканер QR/штрих-кода, подтверждение/опровержение наличия, сверка факт vs эталон, расхождения |
| **Отчёты** | Валовые остатки, выборка по складу, гарантийная аналитика |
| **Списание** | Акт дефектовки → акт списания → авто-задача ответственному на дозакупку |

## Стек

Vite · React · Tailwind CSS · чистый клиент (без бэкенда, данные мок).

## Запуск

```bash
npm install
npm run dev        # http://localhost:5173
npm run build      # production-сборка в dist/
npm run deploy     # публикация на ветку gh-pages
```

## Деплой

GitHub Pages из ветки `gh-pages`. `base` в `vite.config.js` = `/sklad-b24/`.
